# Концептуальное Ядро: «Adaptive Astro-Scheduler»

**Гибридная система планирования жизни на стыке астрономии, AI и персональных данных**

---

## Суть продукта

Это не просто «лунный календарь» с гороскопами, а **инструмент темпоральной оптимизации**. Приложение находит математически идеальные временные окна для конкретных действий пользователя, накладывая астрономические циклы на личный контекст и профессиональные цели.

---

## Архитектура: Система Слоёв (Layered Architecture)

Ядро приложения работает как графический редактор (Photoshop), но для времени. Каждый день — это «холст», на который накладываются слои данных. Итоговая рекомендация — это сумма всех активных слоёв.

### 1. Базовые слои (Hard Science)
Фундамент системы. Это 17+ математически точных астрономических показателей:
- Эфемериды NASA/Swiss Ephemeris
- Фазы Луны
- Синодические циклы
- Планетные часы
- Ингрессии

### 2. Персональный слой (Natal Context)
Статический слой, основанный на времени и месте рождения пользователя (Натальная карта). Он определяет индивидуальный резонанс с общими транзитами.

**Пример:** Полнолуние нейтрально для всех, но критично для вас, так как бьёт по вашему Марсу.

### 3. Гео-слой (Spatial Context)
Динамический слой, пересчитывающий сетку домов гороскопа и Асцендент в зависимости от текущей GPS-локации или планируемой поездки. Позволяет моделировать будущее: «Как изменится моя удача в карьере, если я буду в Берлине на следующей неделе?»

---

## Интеллект: LLM Pipeline и Генерация Правил

Пользователю не нужно знать астрологию. Всю сложность берет на себя AI.

### Input
Пользователь пишет на естественном языке:

> *«Я кардиохирург, у меня сложные операции по понедельникам, и я заметил, что на растущей луне пациенты выздоравливают быстрее».*

### Translation
LLM анализирует профиль и превращает его в программный код (Правило). Создается кастомный слой «Хирургия», который содержит логику:

```
ЕСЛИ (Луна.Растущая) И (Знак != Рак) И (Меркурий.Директный) 
ТО Score += 10
```

### Constraint Satisfaction (CSP)
Когда пользователь спрашивает «Когда лучше оперировать в феврале?», система запускает алгоритм поиска (CSP Solver), который сканирует тысячи часов, проверяет сотни ограничений и выдает 3 идеальных интервала.

---

## Эволюция: Feedback Loop

Система самообучается. Пользователь отмечает реальные результаты:
- ✅ «Операция прошла успешно»
- ❌ «Сделка сорвалась»

Если статистика показывает расхождение с базовыми правилами, система (через AI) корректирует веса алгоритмов персонально для этого пользователя.

---

## Ценность

Приложение превращает мистику и общие фразы в **инженерную задачу оптимизации ресурсов**, позволяя планировать деятельность в моменты наименьшего сопротивления среды и максимальной личной эффективности.

---

## Ключевые Инновации

### 1. Layer Registry Pattern
Позволяет пользователям добавлять кастомные слои без кодинга через 4 интерфейса:
- **Simple** - название + описание
- **Guided** - пошаговые вопросы
- **LLM** - свободный текст → AI генерирует код
- **Code** - прямая реализация на TypeScript

### 2. CSP Solver for Time
Первый в индустрии constraint satisfaction solver специально для планирования деятельности с учётом астро-условий.

### 3. Geolocation Intelligence
- Relocation charts (измени город → новая аstrologia)
- Travel optimizer (лучшие даты И локации для путешествия)
- AstroCartography visualization (Venus lines, Jupiter lines на карте)

### 4. Self-Learning Loop
Основано на реальных исходах, не на философии. Если пользователь говорит «это сработало», система учится.

## Примеры Использования

### Хирург
```
Проблема: Когда оперировать плановых пациентов в феврале?

Входные данные:
- Профиль: Опытный кардиохирург
- Наблюдение: Успех на растущей луне
- Ограничение: Помещения заняты по вторникам
- Критерий: Нулевая смертность

Вывод системы:
5, 12, 19 февраля (89% оптимальности)
```

### Трейдер
```
Проблема: Когда входить в позицию в марте?

Входные данные:
- Профиль: Day trader, 5 лет опыта
- Наблюдение: Лучше всего на новолуние
- Ограничение: Рынок закрыт в выходные
- Критерий: Максимум профита

Вывод системы:
1, 8, 15, 22, 29 марта
```

### Стартапер
```
Проблема: Когда запустить продукт? (планирую на неделю вперёд)

Входные данные:
- Профиль: Entrepreneur, Tech startup
- Наблюдение: Успех на дни благоприятные для коммуникации
- Ограничение: Команда отпускается
- Критерий: Медиа-внимание, word-of-mouth

Вывод системы:
5-11 апреля (92% оптимальности для PR-цикла)
```

## Технический Стек

### Backend
- **Node.js/TypeScript** - основной язык
- **Express/NestJS** - API framework
- **Swiss Ephemeris** - астрономические расчёты

- **Docker** - контейнеризация
